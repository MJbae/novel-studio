# 캐릭터 대화 DNA — 상황 감응형 대화 설계 가이드

캐릭터시트의 비언어 태그가 **몸**의 팔레트라면, 이 문서는 **말**의 팔레트다.
각 캐릭터가 정보를 어떻게 처리하고, 어떤 논리로 말하고, 상황에 따라 어떻게 변주하는지를 정의한다.

episode-rewriter는 대사를 쓸 때 이 문서를 참조하고, character-sculptor는 대사의 DNA 정합성을 진단할 때 참조한다.

---

## 1. 대화 DNA란 무엇인가

비언어 태그가 캐릭터의 신체적 반응을 정의한다면, 대화 DNA는 캐릭터가 **왜, 어떻게** 말하는가를 정의한다.

| 차원 | 비언어 태그 (기존) | 대화 DNA (신규) |
|------|-------------------|----------------|
| 대상 | 몸 — 손, 시선, 자세 | 말 — 어휘, 논리, 리듬 |
| 기능 | 감정을 보여준다 | 사고 방식을 보여준다 |
| 반복 위험 | "또 같은 동작" | "또 같은 방식으로 놀람" |
| 변주 방법 | 5단계 강도 | 상황 유형별 사고 경로 변주 |

**핵심 원칙**: 같은 캐릭터라도, **좋은 소식을 들을 때의 반응**과 **위기를 마주할 때의 반응**은 사고 경로가 다르다. 비언어뿐 아니라 대사의 내용·구조·길이도 달라져야 한다.

## 2. 캐릭터별 대화 DNA 프로필

캐릭터별 대화 DNA 프로필은 novel-config.md에서 지정된 `character_dialogue_dna` 파일에서 로드한다.
해당 파일이 없으면 캐릭터 상세 문서(character_detail)의 캐릭터 프로필을 참조한다.

### 대화 DNA 프로필 작성 템플릿

새 프로젝트가 대화 DNA 파일을 작성할 때 아래 템플릿을 참고한다.

#### 캐릭터 DNA 프로필 양식

```yaml

정보_처리: [새 정보를 받았을 때 처리 방식. 예: "즉시 검산" / "자기 경험에 비추어 이해"]
설득_방식: [이 캐릭터가 설득되는 방법. 예: "데이터로" / "감정적 동의 먼저"]
감정_누출: [감정이 드러나는 순간. 예: "가족 이야기가 나올 때"]

대사_DNA:
  구조: [길이, 문장 특징. 예: "짧다(10~25자). 주어 생략 빈번."]
  어휘: [사용하는 언어 특징. 예: "추상어 금지. 구체적 수치만."]
  특징: [고유한 대사 습관. 예: "말 중간에 계산을 삽입한다."]
  금지: [절대 하지 않는 것. 예: "감탄사, 자기 자랑."]
```

## 3. 상황 유형별 캐릭터 반응 매트릭스

같은 사건에 여러 캐릭터가 동시에 반응할 때, **전원 동일 반응 금지** 원칙을 적용하기 위한 매트릭스.

프로젝트별 character_dialogue_dna 파일에 캐릭터별 반응 매트릭스를 정의한다.

**작성 원칙**: 3명 이상 동시 반응 시, 매트릭스에서 **다른 열**의 반응을 선택한다.
놀라움만 3명 --> 놀라움 + 검산 + 보류 등으로 분화한다.

## 4. 대화 반복 방지 — 변주 기법

### 4-1. 같은 캐릭터의 대사 단조로움 방지

같은 캐릭터가 한 에피소드 안에서 같은 방식으로만 반응하면 단조롭다.

**규칙: 한 화 안에서 같은 사고 경로 2회까지.** 3회째는 다른 경로로 반응.

### 4-2. 정보 전달 대사의 캐릭터화

순수 정보 전달이 필요한 대사도 캐릭터 DNA를 통과시킨다.

**원칙**: 정보를 전달하되, 그 정보가 **이 캐릭터의 삶에서 무엇을 의미하는지** 한 줄 덧붙인다.

### 4-3. 화간 대화 변주

직전 2화에서 같은 캐릭터가 같은 사고 경로를 사용했으면, 이번 화에서는 다른 경로를 선택한다.

## 5. 서술체 속의 대화 — 웹소설 기법

### 5-1. 대화는 장면의 일부다

대화문은 독립적으로 존재하지 않는다. **공간·시선·행동의 흐름 안에서** 발화된다.

```
나쁜 예 (대본식):
캐릭터A: "수치가 맞아?"
캐릭터B: "네."
캐릭터A: "정말?"

좋은 예 (서술체):
캐릭터A가 정산서를 손가락으로 짚었다. 숫자 아래를 두 번 두드렸다.
"수치가 맞다고?"
수첩을 펼쳐 보였다.
"첫 수확입니다."
캐릭터A의 손이 멈췄다. 정산서 위에 놓인 채.
```

**핵심**: 대사 전후에 **시선·손·공간**이 있다. 대사만 나열하면 대본이다.

### 5-2. 대사 사이의 서술 — 호흡 만들기

대사와 대사 사이에 서술을 넣되, 그 서술이 **다음 대사의 무게를 준비**한다.

### 5-3. 침묵을 서술하는 법 — "침묵했다" 대신

"침묵했다"는 가장 게으른 침묵 묘사다. 침묵의 **질감**이 있어야 한다.

| "침묵했다" 대신 | 전달되는 뉘앙스 |
|---------------|--------------|
| 공기가 무거워졌다. | 긴장 |
| 커피가 식을 동안 아무도 서류를 건드리지 않았다. | 충격 후 정적 |
| 창밖에서 소리만 들렸다. | 적막 |
| 펜이 수첩 위에서 멈춘 채였다. | 결단 직전 |

### 5-4. 대화 리듬 — 짧은 대사와 긴 서술의 교차

```
[짧은 대사]  "수치."
[긴 서술]    서류 위에서 시선이 숫자를 따라갔다. 한 줄, 두 줄.
[짧은 대사]  감탄.
[짧은 서술]  고유 동작.
[긴 대사]    개인적 맥락이 담긴 반응.
```

## 6. 진단 기준 (character-sculptor 참조용)

### DIALOGUE 진단 체크리스트

| 항목 | 기준 | 방법 |
|------|------|------|
| DNA 정합 | 캐릭터의 사고 패턴이 대사에 반영되는가? | character_dialogue_dna 파일의 대사 DNA와 대조 |
| 상황 감응 | 상황 유형에 따라 대사 경로가 달라지는가? | 반응 매트릭스와 대조 |
| 화 내 변주 | 같은 사고 경로가 3회 이상 반복되지 않는가? | §4-1 |
| 화 간 변주 | 직전 2화와 같은 사고 경로가 3화 연속이 아닌가? | §4-3 |
| 정보의 캐릭터화 | 정보 전달 대사에도 개인 맥락이 실려있는가? | §4-2 |
| 서술 통합 | 대사가 서술 흐름 안에 있는가? (대본식 금지) | §5-1 |

### 점수 기준

| 점수 | 의미 |
|------|------|
| 9~10 | 모든 대사가 교체 불가능. 이 캐릭터만의 사고 패턴이 드러남. 상황별 변주 자연스러움. |
| 7~8 | 대부분 정합하나, 1~2곳에서 기능 대사(정보만 전달)가 있음 |
| 5~6 | DNA는 맞지만 변주가 없음. 같은 사고 경로만 반복 |
| 3~4 | 다른 캐릭터와 교체 가능한 대사 다수. 개성 미약 |
| 1~2 | DNA와 전혀 다른 대사. 완전 재작성 필요 |
