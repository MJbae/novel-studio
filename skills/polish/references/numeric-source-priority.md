# 수치 교차검증 정본 우선순위 (Numeric Source Priority)

모든 에이전트(story-analyst, revision-executor, revision-reviewer, quality-verifier, episode-architect)가 수치/시간 불일치를 판단할 때 **동일한 우선순위**를 적용한다.

---

## 정본 우선순위 (높은 순서)

| 순위 | 정본 | 키 | 적용 범위 | 설명 |
|------|------|-----|----------|------|
| **1** | 플롯가이드 EP별 확정값 | `{PLOT_DOC}` | 해당 EP에 명시된 수치 | 가장 구체적. EP별 테이블에 기재된 수치가 최우선 |
| **2** | 부트스트랩 매크로 수치 | `{BOOTSTRAP}` | 세계관 전체 규칙 | 플롯가이드에 없는 매크로 수치(세계 설정, 경제 규칙 등) |
| **3** | 검증 완료 수치 | `{VERIFY}` | 검증 기록 | 이전 윤문/검증에서 확정된 수치 기록 (있을 경우) |
| **4** | 직전 에피소드 | `EP{N-1}.md` | 서사 연속성 | 이전 에피소드에 명시된 수치 (서사 흐름 유지용) |

---

## 특수 규칙

### 캐릭터 나이
- **항상** `{CHAR_CORE}`의 생년(출생년도)을 기준으로 계산한다
- 나이 = (해당 EP 시점 연도) - (생년)
- 플롯가이드/부트스트랩보다 생년 기준 계산이 우선한다
- `{CHAR_CORE}`에 생년이 없으면: `[나이미정]` 태그를 달고, 다른 문서에서 추론한 값에 `[추론]` 표시

### 충돌 해소 규칙
1. **상위 정본 우선**: 순위 1이 순위 2와 충돌하면 순위 1을 따른다
2. **산술 검증**: 수치를 선택한 후, 같은 EP 내 관련 산술(합산/나눗셈/비율)을 직접 계산하여 정합성을 확인한다
3. **파급 확인**: 수치를 변경하면, 같은 EP 내에서 해당 수치를 참조하는 모든 문장을 찾아 함께 수정한다
4. **양쪽 모순 시**: 정본 순위로도 판단이 어려우면 양쪽 값을 모두 보고서에 기록하고 `[충돌미해소]` 태그를 단다

### 시간 마커 처리
1. **상대 시간 → 절대 시간 변환**: "내일", "3일 후", "다음 주" 등은 해당 장면의 명시적 날짜를 기준으로 절대 날짜로 변환한다
2. **기준 날짜 결정**: 장면 내 명시적 날짜 > 직전 장면의 명시적 날짜 > 타임라인 락(episode-architect) > 부트스트랩 시간선
3. **기준 날짜 부재 시**: `[시간기준미상]` 태그를 달고, 가장 가까운 명시적 날짜에서 상대 계산

## 에이전트별 적용 방법

| 에이전트 | 적용 시점 | 행동 |
|---------|----------|------|
| **episode-architect** | 설계도 작성 시 | 확정 수치란에 정본 출처를 병기. 충돌 시 우선순위에 따라 결정 |
| **story-analyst** | 3단계 프로토콜 수행 시 | 추출한 수치를 우선순위에 따라 교차대조. 결과를 표로 출력 |
| **revision-executor** | TIMELINE/NUMBER 교정 시 | 직전 2화 grep → 정본 우선순위 순서로 정본 확인 → 수정 |
| **revision-reviewer** | 최종 검증 시 | 교정된 수치가 우선순위를 따르는지 재검증 |
| **quality-verifier** | CREATE/REWRITE 검증 시 | 3단계 프로토콜로 추출한 값을 우선순위에 따라 판정 |

## novel-config.md 연동

novel-config.md의 "수치 교차검증 정본 우선순위" 섹션은 이 문서와 동일한 우선순위를 참조한다.
프로젝트별로 추가 정본을 삽입할 경우, novel-config.md에 명시하고 이 문서의 순위 체계 내에 위치를 지정한다.
