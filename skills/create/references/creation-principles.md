# 에피소드 창작 원칙 상세

episode-creator 에이전트가 상세한 집필 가이드가 필요할 때 참조하는 문서.

## 1. 장면 구성 원칙

### 장면 시작
- 시공간 앵커로 시작: 시간(구체적 시각 또는 상대 시간) + 장소(구체적 위치)
- 감각 디테일 1-2개로 분위기 설정 (시각/청각/촉각/후각 중 택)
- 직전 장면과의 시간 간격이 명확해야 함

### 장면 전개
- 한 장면에 하나의 핵심 갈등/목표만 다룬다
- 장면 내 긴장 곡선: 도입(30%) → 전개(40%) → 절정/전환(30%)
- 정보 전달은 대화 또는 행동을 통해 (서술자 설명으로 직접 전달 금지)

### 장면 마무리
- 다음 장면으로의 전환 동기를 남긴다 (인물의 결심, 새 정보, 시간 제한)
- 마지막 장면은 반드시 미해결 상태로 끝난다

### 장면 수와 리듬
- 에피소드당 2-4장면이 적절하다. 5장면 이상이면 산만해진다
- 긴장 장면과 이완 장면을 교대한다 (연속 3장면 고긴장 금지)
- 각 장면의 길이는 균등할 필요 없다. 클라이맥스 장면에 분량을 집중한다

## 2. 대화 작성 원칙

### 대화 리듬
- 3문장 이상 연속 대화 후에는 비언어 행동이나 내면 묘사를 삽입한다
- 대화 태그는 "말했다" 계열 대신 행동 태그를 선호한다
  - 나쁜 예: "그가 말했다"
  - 좋은 예: "서류를 내려놓으며"
- 한 대화씬에서 같은 행동 태그를 2회 이상 쓰지 않는다

### 정보 전달 대화
- 전문가물에서 정보를 전달할 때: 한 번에 3문장 이상의 설명을 하지 않는다
- 청자의 반응(질문, 의문, 감탄)을 삽입하여 독자의 이해를 돕는다
- 전문 용어 사용 시 맥락으로 자연스럽게 의미를 전달한다 (각주식 설명 금지)

### 갈등 대화
- 대립하는 두 인물의 대화: 한 쪽이 3연속 우위를 점하지 않는다
- 밀고 당기기 구조: 공격 → 방어 → 반격 패턴
- 감정 에스컬레이션: 대화가 진행될수록 감정 온도가 서서히 올라간다

### 침묵과 비언어
- 침묵은 캐릭터시트의 침묵 예외 인물(아버지 등)을 제외하고 비언어로 교체한다
- 비언어 표현: 시선(응시/회피), 자세(기댐/경직), 습관적 행동(손가락 두드리기 등)
- 인물별 고유 비언어 팔레트를 캐릭터시트에서 확인한다

## 2.5. 자연스러운 한국어 문체 가이드

### 근본 원칙: "서술자는 투명해야 한다"
- 서술자가 독자에게 "이건 중요해", "이건 슬퍼", "이건 놀라워"라고 신호를 보내면 안 됨
- 장면을 보여주고, 독자가 스스로 감정을 느끼게 해야 함
- AI의 근본 문제: 감정을 "해설"하려는 경향. 한국어 웹소설은 행동과 대사로 감정을 전달

### AI 문체 자가진단 체크리스트
(문장 작성 후 스스로 확인)
1. 이 문장을 삭제해도 장면이 전달되는가? → 삭제
2. 감정 명사(공포, 경악, 체념, 기쁨)가 주어인가? → 신체 반응으로 교체
3. 서술자가 상황을 "정리"하고 있는가? → 삭제하고 행동/대사로 교체
4. 같은 의미를 다른 비유로 2번 이상 썼는가? → 더 강한 한 문장만 남기기
5. "~인 것이다", "~는 셈이었다", "~라고 할 수 있었다" → 직접 서술로 교체
6. 마지막 문장이 "그렇게 ~했다", "새로운 ~이 시작되었다" 류인가? → 대사/행동으로 교체

### 서술자 개입 금지 패턴 (구체적)
| 패턴 | 이유 | 대안 |
|------|------|------|
| "그것은 ~이었다" | 서술자가 해석 강요 | 장면으로 보여주기 |
| "~라는 것을 알았다/깨달았다" | 감정 해설 | 행동 변화로 인지 표현 |
| "~가 아니라 ~였다" | 대비 구문 남용 | 한 문장으로 직접 서술 |
| "바로 그 순간/그때" | 메타서술 신호 | 삭제, 행동으로 시점 표시 |
| "~하는 것은 ~때문이었다" | 인과 해설 | 원인→결과를 장면 순서로 보여주기 |
| 감정 명사 + "가 밀려왔다/올라왔다" | 감정 해설 | 신체 반응(떨림, 호흡 변화 등) |
| "새로운 ~이 시작되었다" | 메타서술 엔딩 | 대사 또는 행동으로 종결 |

### 비유 밀도 제한
- 한 문단에 비유 1개 이하
- 3문단 연속 비유 금지
- 비유는 인물의 직업/경험에서 나와야 함 (연금 전문가가 "기하급수적 감소"라고 비유하는 건 자연스러움)

### 에피소드 제목 원칙
- 1~2단어. 명사 또는 명사구.
- 수식어, 조사, 관형절 제거.
- 예: "고갈", "명찰", "첫 상담", "경고"

## 3. 훅 배치 전략

### 오프닝 훅 (첫 1000자)
- 사건 직접 투입: 독자가 3줄 이내에 "무슨 일이 벌어지고 있는지" 파악해야 한다
- 금지: 아침 기상, 회상, 세계관 설명, 이전화 요약
- 효과적 기법:
  - 대화 중간 시작 (이미 진행 중인 상황에 독자를 던짐)
  - 위기 상황 직입 (시간 제한, 실패 직전, 갈등 폭발)
  - 의외의 정보 공개 (독자가 모르던 것을 인물이 알고 있음)

### 중반 훅 (50% 지점)
- 반전 또는 새로운 정보 공개로 독자의 기대를 재설정한다
- "알고 있던 것이 사실이 아니었다" 패턴이 가장 강력하다
- 기법:
  - 새 인물의 등장과 기존 판도 변화
  - 숨겨진 조건/제약의 드러남
  - 동맹의 균열 또는 적의 의외의 약점

### 엔딩 훅 (마지막 500자)
5단계 클리프행어 (상황에 따라 선택):
1. 약한: 새로운 계획 선언 ("다음엔 이렇게 하겠다")
2. 보통: 예상치 못한 인물 등장
3. 강한: 중요 정보 부분 공개 (전체가 아닌 일부만)
4. 매우 강한: 위기 직면 (시간 제한, 배신, 실패)
5. 최강: 세계관/인물 설정을 뒤집는 정보 공개

## 4. 전문가물 정보 통합

### 전문 지식 노출 원칙
- 독자는 전문 지식 자체가 아니라, 그 지식으로 인한 **상황 역전**에 흥미를 느낀다
- 지식 노출 패턴: 위기 → 전문 지식 적용 → 상황 역전 → 주변인 경탄
- 한 에피소드에 전문 지식 핵심 활용은 1-2회로 제한 (과다 노출 = 강의형 서술)

### 수치 자연스럽게 삽입
수치는 대화나 상황 속에서 자연스럽게 노출한다:
- 나쁜 예: "그 토지의 면적은 1만 2천 평이었다."
- 좋은 예: "'만이천 평이요?' 부장이 서류를 다시 들여다봤다."
- 나쁜 예: "투자금은 3억 원이 필요했다."
- 좋은 예: "삼억." 이서연이 손가락 세 개를 펴 보였다. "그 이상은 못 끌어."

### 회귀자의 미래 지식
- 미래 지식 활용 시 "왜 아는지"에 대한 위장 논리를 반드시 포함한다
- 직접 "미래에서 왔으니까"라고 생각하는 독백은 독자 공감을 깨뜨린다
- 대신: 관련 자료를 사전에 준비한 것처럼 보이는 장치, 직감으로 위장, 분석력으로 포장

## 5. 한국 플랫폼 최적화

아래 원칙은 문피아, 네이버시리즈, 카카오페이지, 리디, 조아라, 노벨피아에 공통으로 적용한다.
세부 기준은 프로젝트의 `target_platform`에 맞는 `_workspace/platform-guide-{플랫폼}.md`를 우선 참조한다.

### 모바일 가독성
- 문단: 3-5문장 (긴 문단은 분리)
- 문장: 40자 이내 권장, 60자 초과 금지
- 대화와 서술의 교대: 연속 서술 5문장 이하

### 사이다/고구마 밸런스
- 사이다 장면(독자가 "아 시원하다"고 느끼는 장면)은 에피소드당 최소 1회
- 고구마 장면(답답한 상황)은 2화 이상 연속 금지
- 고구마 후 사이다의 낙차가 클수록 독자 만족도 상승

### 포인트 장면
- 주인공이 상대를 압도하는 장면: 과정을 보여준다 (결과만 보여주면 맥이 빠짐)
- 상대의 표정/반응을 통해 주인공의 대단함을 간접적으로 전달한다
- "보여주기(showing)" 원칙: 독자가 스스로 "와, 대단하다"고 느끼게 만든다

### 유료 전환 구간
- 유료 전환 직전 에피소드(플롯가이드에 명시)는 훅 강도를 최대로 설정한다
- 무료 구간에서도 유료 수준의 품질을 유지하여 독자 신뢰를 구축한다
- 유료 전환 직후 에피소드는 즉시 보상(사이다)을 제공하여 결제 만족도를 높인다
